# UX レビューレポート — 意思決定マトリクスツール

**レビュー日**: 2026-05-19
**レビュアー**: UXレビューエージェント（ux-reviewer）
**対象**: `apps/decision-matrix/`

---

## ペルソナ設定

- **属性**: フリーランス／会社員、30〜40代
- **シーン**: 転職先を2〜3社で迷っている、または案件Aと案件Bで悩んでいる
- **ITリテラシー**: 普通（スプレッドシートは使える。「加重スコアリング」という言葉は知らない）
- **デバイス**: 主にスマートフォン（iPhone/Android）
- **動機**: 「感覚に頼らず、数字で整理して後悔のない選択をしたい」

## 観点別レビュー

---

### 1. ファーストビュー（h1・説明文・CTAが明確か）

**評価: ⚠️ 要改善**

**良い点**
- h1 に「加重スコアリング」「最適解を可視化」というキーワードが含まれており、SEO意識が反映されている。
- サブタイトルで「転職・案件選択・ツール選定」という具体的な用途が示されており、ターゲットユーザーに刺さりやすい。
- 「登録不要・ブラウザだけで完結」という安心感のあるメッセージがある。

**問題点**
- h1 の「加重スコアリングで最適解を可視化」という表現は、ITリテラシーが普通のユーザーには難解。「加重スコアリング」という専門用語を初見で理解できるユーザーは少ない。
- ファーストビューに「始め方」や「今すぐ試す」といった明確なCTAボタンがない。ページを開いたときに「何をすればいいか」が一見でわからない。
- プリセットボタン（「転職先比較」「案件比較」「ツール選定」）がヘッダー直下に来るのは良いが、「まずプリセットを押してください」という誘導テキストが弱い。section-desc の「サンプルデータで使い方を確認できます」は目立たない（font-size: 0.8rem、色 #64748b と薄い）。
- スマホでは h1 フォントが 1.15rem（モバイル）と小さく、ヘッダー全体のインパクトが弱い。

**改善提案**
- h1 の表現を「比較で迷ったら数字で決める — 意思決定マトリクスツール」のようにユーザー目線のコピーに変更する。
- ヘッダー直下に「まず転職先比較のサンプルを試してみる」などのCTAボタンを1つ配置する。

### 2. 入力UX（評価基準・選択肢の追加操作がわかりやすいか）

**評価: ✅ おおむね良好（軽微な改善余地あり）**

**良い点**
- プリセット機能（「転職先比較」「案件比較」「ツール選定」）により、初回ユーザーがゼロから入力しなくてもサンプルで動作確認できる設計は非常に優秀。
- 「+ 基準を追加」「+ 選択肢を追加」ボタンが全幅の破線ボタンで視認性が高い。
- 各入力欄のタッチターゲットが `min-height: 44px` で確保されており、スマホ操作に配慮されている。
- 削除ボタン（×）も `min-height: 44px / min-width: 44px` 確保済み。
- 基準追加時・選択肢追加時に最後の入力欄へ自動フォーカスが移動するため、連続入力がしやすい。
- 最大項目数（基準10、選択肢8）に達したらボタンが `disabled` になり、これ以上追加できないことが示される。

**問題点**
- 重みの入力欄（`input-weight`）が `type="number"` の数値入力のみ。スマホでは数字キーボードが表示されるが、「1〜10の範囲」という制約が視覚的に見えにくい。section-desc には「重み（1〜10）を入力してください」とあるが、入力欄自体にプレースホルダーや補足がない。
- スコア入力マトリクスの各セル（`score-input`）の幅が 3.2rem（モバイル）と小さく、スマホでタップが難しい。`min-height: 40px` は 44px を下回っており基準未達。
- 評価基準と選択肢を追加したあと、スコアマトリクスのどのセルを埋めればいいかが直感的にわからない。テーブルに「ここにスコアを入力してください」などのガイドが一切ない。
- 基準が1つしかない場合に削除ボタンが押されても何も起きない（`if (state.criteria.length <= 1) return;`）が、押しても反応がなくユーザーが困惑する可能性がある。エラーメッセージや disabled 化が望ましい。

**改善提案**
- `score-input` の `min-height` を 40px から 44px 以上に修正する。
- スコアマトリクスの上に「各セルに 1〜10 のスコアを入力してください」という案内を追加する。
- 基準・選択肢が最低1つのときは削除ボタンを `disabled` にするか、視覚的に非表示にする。

### 3. 結果表示（スコア・ランキング・推奨表示が直感的か）

**評価: ✅ 良好**

**良い点**
- スコア入力と連動してリアルタイムで結果が更新される設計は直感的で快適。
- 1位に「推奨」バッジ + `result-top` スタイル（青背景・ボーダー）で視覚的に強調されており、最適解が一目でわかる。
- CSS バーチャートでスコアの相対比較が視覚的に把握できる。
- 「1st」「2位」「3位」の順位表示と加重合計スコアの数値が大きく表示されており、スマホでも見やすい。
- `aria-live="polite"` 付きのコンテナで、スクリーンリーダーへも配慮されている。

**問題点**
- スコアが全員同じ場合（全て5点）の初期表示で、順位が意味をなさないにもかかわらず「推奨」バッジが表示される。「スコアが同点のため推奨なし」の状態表示があるとより親切。
- 「加重合計」という表示ラベルが `font-size: 0.7rem` と非常に小さく、スマホで読めない可能性がある。
- スコアの最大理論値（全基準10点×全重み合計）が示されていないため、「480点」という数値が良いのか悪いのか文脈がない。パーセンテージ表示や「最高〇点中」の補足があると理解しやすい。

### 4. エラー処理（未入力・不正値への対応）

**評価: ⚠️ 要改善**

**良い点**
- `clampInt()` 関数により、スコアと重みは範囲外の値が自動的に最小値にクランプされる（バリデーションロジックは存在する）。
- localStorage の読み込み失敗を `try/catch` で握りつぶしており、クラッシュしない。

**問題点**
- スコア入力欄に範囲外の値（例: 0、11、-1、テキスト）を入力しても、ユーザーへのフィードバックが一切ない。`clampInt` でサイレントに 1 にクランプされるが、ユーザーは入力欄の見た目上は 0 のままで実際の計算では 1 が使われるという不一致が生じる。
- 評価基準名が空のまま計算が走る（テーブルヘッダーに「（未入力）」と表示されるが、スコア計算は問題なく実行される）。ユーザーが気づかないまま無名の基準でスコアを入れてしまう可能性がある。
- 重み入力欄に非数値や空欄を入力した場合、`clampInt` が `min`（1）を返すが、入力欄自体は空欄のまま表示される視覚的不整合がある。
- リセットボタンは `confirm()` ダイアログを使っているが、iOSの一部ブラウザでは `confirm()` がブロックされる場合がある。

**改善提案**
- スコア・重み入力時に範囲外の値が入力されたら、入力欄にエラーボーダー（赤）を表示し、`1〜10の範囲で入力してください` のツールチップまたはインラインメッセージを出す。
- `input` イベントの後に入力欄の `value` を `clampInt` の結果で書き戻す（現在は `state` の値だけ更新し DOM は更新していない箇所がある）。

### 5. モバイル対応（viewport, maximum-scale=1.0, タッチターゲット44px以上）

**評価: ⚠️ 要修正（1点の基準未達）**

**良い点**
- `<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0">` が設定されており、iOSの自動ズームを防止している。
- ほぼすべてのインタラクティブ要素（ボタン・テキスト入力）で `min-height: 44px` が確保されている。
- `.table-scroll` に `overflow-x: auto` と `-webkit-overflow-scrolling: touch` が設定されており、横スクロールテーブルがスマホで操作可能。
- `flex-wrap: wrap` によりプリセットボタンが小さな画面でも折り返し表示される。
- モバイルファーストのメディアクエリ（min-width: 640px）で段階的にフォントサイズを拡大する設計になっている。

**問題点（要修正）**
- `score-input`（スコアマトリクスのセル入力欄）の `min-height: 40px` がタッチターゲット基準の 44px を下回っている。スマホで基準値の入力欄が小さくてタップしにくい。
- スコアマトリクスで列が多い場合（基準5〜10個）、横スクロールが必要になるがスクロール可能であることのヒント（スワイプ誘導）が視覚的に存在しない。

**改善提案**
- `.score-input` の `min-height` を 44px 以上に修正する。

### 6. 信頼性（プライバシー、データ保存説明）

**評価: ✅ 良好**

**良い点**
- `howto-section` 内に「プライバシーについて」セクションがあり、「入力したデータはすべてお使いのブラウザ内（localStorage）にのみ保存されます。外部サーバーへの送信は一切行いません」と明記されている。
- フッターに「プライバシーポリシー」へのリンクが存在する。
- 「登録不要」「ブラウザだけで完結」という安心感のある文言がヘッダーに掲載されている。
- GA4トラッキングコード（G-DK2WQM49L6）が正しく `<head>` 内に実装されている。

**問題点**
- プライバシーに関する説明が「使い方」セクションの下部に埋もれており、ファーストビューから非常に遠い位置にある。初めてのユーザーが個人情報漏えいを懸念してページを離脱する前にこの情報を見つけられない可能性がある。
- 「ブラウザのキャッシュ/LocalStorageをクリアするとデータが消える」という注意書きがない。

**改善提案**
- ヘッダー説明文やマトリクス入力セクションの近くに「入力データはブラウザのみに保存。外部送信なし。」の1行を追加する。

### 7. コンテンツ（使い方セクション、解説の充実度）

**評価: ✅ 良好**

**良い点**
- 「使い方」セクションが6ステップの番号付きリストで構成されており、手順が明確。
- 「加重スコアリングとは」の解説で計算式（加重合計 = スコア × 重みの合計）と具体例（給与 8×7=56）が示されており、非専門家にも理解しやすい。
- 「活用シーン」で転職・フリーランス案件・SaaS・引越し・投資という5つのユースケースが示されており、ターゲットユーザーに刺さりやすい。
- SEO_REPORT で推奨されていた「300〜500字の活用シーン解説」が実装されている。

**問題点**
- 「使い方」セクションがページの最下部（広告・アフィリエイトブロックの直前）にあり、スクロールしないと見えない。初めてのユーザーはプリセットボタンを押す前に迷う可能性がある。
- ステップ3「スコア入力マトリクス」の説明が「各セルに1〜10のスコアを入力します」のみで、「高いスコア = その基準においてその選択肢が優れている」という採点基準の説明がない。ユーザーが逆向きにスコアを入れてしまうリスクがある（例: 「通勤が遠い = 悪い = 低スコアを入れる」という理解が必要だが説明がない）。
- SEO_REPORT で推奨されていた「CSV書き出し・コピー機能」「レーダーチャート」が未実装。差別化機能として今後の優先課題。

### 8. 広告・アフィリエイト（ad-unit配置、アフィリエイトブロック）

**評価: ✅ 良好**

**良い点**
- 広告ユニットが2箇所（ヘッダー直下 `#ad-top`、結果セクション上 `#ad-result`）に配置されており、ユーザーのコンテンツ閲覧フローを大きく阻害しない位置にある。
- `aria-hidden="true"` が設定されており、スクリーンリーダーに広告が読み上げられない配慮がある。
- アフィリエイトブロック（FREENANCE）に「PR」ラベルが表示されており、ステルスマーケティング規制への対応が取れている。
- アフィリエイトリンクに `rel="nofollow"` が設定されており、SEOへの悪影響を防いでいる。
- FREENANCE（フリーランス向け損害賠償保険）はターゲットユーザー（フリーランス・会社員）と関連性が高く、コンバージョン率が期待できる選択。
- アフィリエイトリンクの `min-height: 44px; line-height: 44px` でタッチターゲットが確保されている。

**問題点**
- `#ad-top`・`#ad-result` の `.ad-unit` 要素は現時点ではプレースホルダー（グレーの空ブロック）のまま。AdSense審査前の状態と思われるが、`min-height: 90px` の空の灰色ブロックがユーザーには「壊れた要素」に見える可能性がある。
- AdSense 未承認段階では、プレースホルダーの `display: none` またはダミーテキスト（「広告」など）の表示が望ましい。

## 総合判定

**⚠️ 要修正**

ブロッカー級の問題はなく、ツールとしての基本機能・情報設計・SEO対応は高い水準で実装されている。ただし、以下の修正を完了してからリリースすることを推奨する。

### 必須修正（リリース前に対応）

1. **`score-input` の `min-height` を 40px から 44px 以上に変更**（モバイルタッチターゲット基準未達）
2. **不正値入力時のフィードバック追加**（DOM の value を clampInt 結果で書き戻す + エラー表示）
3. **ファーストビューの導入文を平易な表現に変更**（「加重スコアリング」をサブ表現として扱う）

### 推奨修正（品質向上）

4. 削除ボタンが最小項目数で押された場合は `disabled` にする or 視覚的フィードバックを追加
5. スコアマトリクスのセルに「スコアが高いほど優れている」という採点基準の説明を追加
6. プライバシー説明（「入力データはブラウザのみ保存」）をヘッダー近くにも表示
7. 広告プレースホルダーを AdSense 承認まで非表示（`display: none`）にする

### 将来対応（差別化強化）

8. CSV書き出し機能（SEO_REPORT で推奨）
9. レーダーチャートによる視覚化（SEO_REPORT で推奨）
10. 結果の同点処理（同スコア時の推奨バッジ非表示 or 複数推奨表示）
