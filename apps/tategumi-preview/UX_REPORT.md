# UX_REPORT — 縦書きプレビュアー

レビュー日: 2026-05-22
レビュアー: UX エージェント（一般ユーザー視点）

---

## ペルソナ設定

- **属性**: カクヨム・小説家になろうに投稿している 20〜30 代
- **目的**: テキストエディタで書いた原稿が縦書きでどう見えるか確認したい
- **ITリテラシー**: 普通（Webサービスは日常的に使うが、開発知識はない）
- **使用端末**: PC・スマホ両方

---

## 8観点レビュー

---

### 1. ファーストビュー（h1・説明文・入力エリアの分かりやすさ）

**評価: ✅ 良好**

- h1「縦書きテキスト・小説原稿プレビュアー — 原稿用紙換算・字数カウント無料」は長いが、キーワードが詰まっており SEO と一致している。ユーザーが「これは自分が探しているツールだ」と即座に判断できる。
- subtitle「小説・エッセイを縦書きでリアルタイム確認。カクヨム・なろう投稿前の仕上がりチェックに。」はペルソナに直撃する訴求文で優秀。
- ページを開くと入力エリアがファーストビューに収まっており、「何をすればよいか」が一目で分かる。
- テキストエリアの placeholder が「ルビは《》記法で指定できます」まで説明しており、機能の存在を自然に伝えている点は良い。

**軽微な懸念点**:
- ヘッダー h1 は PC では 1.3rem に拡大されるが、モバイルでは 1.1rem と小さい。タイトルとして視認性がやや弱い（1.2rem 程度への引き上げを推奨）。

---

### 2. 入力UX（テキストエリアの使いやすさ・サンプルテキスト・設定オプションの分かりやすさ）

**評価: ⚠️ 要確認（軽微）**

**良い点**:
- テキストエリアは 300px（モバイル）/ 400px（PC）と十分な高さがある。
- `resize: vertical` によりユーザーが自分で高さを調整できる。
- 設定セレクトボックスはすべて `min-height: 44px` 対応済み。
- 「サンプルテキストを挿入」ボタンは色がコンテナから浮いており視認性が高く、モバイルでは自動的にプレビュータブへ切り替わる配慮が良い。
- サンプルテキストは夏目漱石「吾輩は猫である」冒頭でルビ付きのため、ルビ機能を同時に体験できて一石二鳥。

**懸念点**:
- 設定パネルが入力エリアの**上**に配置されている。初見ユーザーは「まず入力して → 後から設定を調整する」導線で使うため、設定パネルが先頭にあると若干戸惑う可能性がある。ただし実害は小さい。
- 「1行の文字数」オプションは小説投稿サイトとの関連が分かりにくい。例「カクヨム等の縦書き1行の目安は25〜35文字」などの補足ツールチップまたは説明文があるとより親切。
- 設定変更時に `applyCharsPerLine()` が `height = chars × fontSize × 1.8` の固定計算でプレビューエリアの高さを書き換えるが、行間設定（lh-narrow / lh-wide）が変わったときに `applyCharsPerLine()` が再度呼ばれない。行間を「広め」に変更すると高さ計算がずれる場合があり、ユーザーが見えるはずの文字が途中で切れる恐れがある。→ `lineHeightSel` の change イベントでも `applyCharsPerLine()` を呼ぶべき。

### 3. 結果表示（縦書きプレビューの見やすさ・字数カウント・原稿用紙換算の表示）

**評価: ✅ 良好（1点要修正）**

**良い点**:
- 縦書きに `writing-mode: vertical-rl` + `direction: rtl` を使用しており、右から左へ列が並ぶ正しい縦組み表現。
- 背景色 `#fdf8f0`（和紙色）は縦書き小説の雰囲気に合っており、視認性と情緒の両立ができている。
- ルビ表示（`<ruby><rt>`）は縦書きに対応しており、ルビサイズ 0.5em は適切。
- 字数カウントパネルに「総文字数 / 実質文字数 / 原稿用紙 / 行数」の 4 指標が並び、投稿サイト向け確認として必要十分。
- `aria-live="polite"` 付きで字数カウントがスクリーンリーダーにも対応。

**要修正点**:
- プレビューエリアの高さが固定 500px（モバイル）/ 560px（PC）でありながら、`tategumi-area` の高さが `applyCharsPerLine()` によって動的に書き換えられる。しかし `tategumi-scroll-outer` は `overflow: hidden` のため、エリアがスクロール外枠より背が高くなるとはみ出して消える。縦書きのスクロールは横方向（overflow-x: auto）で実現されているが、エリア高さが `scroll-outer` の高さ 500/560px を超えると下が切れる問題がある。chars=40 かつ fontSize=18 の場合 `40 × 18 × 1.8 = 1296px` となり外枠を大幅超過する。→ `applyCharsPerLine()` の height 計算上限を `scroll-outer` の高さ以内に抑えるか、設計を見直す必要あり。
- 字数カウントの「行数」はテキストの改行数を返しているが、縦書きプレビューの実際の「列数」とは異なる。ユーザーが混乱しないよう「段落数」などのラベル変更を検討。

### 4. エラー処理（空テキスト・長すぎるテキストへの対応）

**評価: ⚠️ 要修正**

**良い点**:
- 空テキスト時は縦書きプレースホルダーが表示され、「何も入力されていない」状態が分かる。
- 字数カウントはすべて 0 表示で整合している。

**要修正点**:
- 非常に長いテキスト（例: 長編小説 10 万字以上）をペーストした際の対応がない。`innerHTML` の更新は同期処理のため、ブラウザがフリーズする恐れがある。最低限、文字数上限（例: 10 万字）のチェックと警告表示を追加すべき。
- XSS 対策として `convertRuby()` 内でテキストを HTML エスケープしてから `innerHTML` に渡す実装は適切。ただし `convertToParagraphs()` の返り値をそのまま `innerHTML` に代入している点は、将来的なコード変更時のリスクとして認識しておく必要がある（現状は安全）。
- 「サンプルテキストを挿入」は既に入力済みのテキストを上書きする。戻す手段（Ctrl+Z は使えるがボタンは未実装）がないことを暗黙で想定している。既存テキストがある場合は確認ダイアログを出すと親切。

---

### 5. モバイル対応（viewport, maximum-scale=1.0, タッチターゲット44px以上、タブUI）

**評価: ✅ 良好**

- `maximum-scale=1.0` を設定済みで iOS の自動ズーム防止対応あり。
- モバイルタブ（入力 / プレビュー）は `min-height: 44px` 対応済み。
- セレクトボックス・サンプルボタン・アフィリエイトリンクもすべて `min-height: 44px` 確認済み。
- タブ切り替えは `aria-role="tablist"` / `aria-selected` / `aria-controls` まで実装されており品質が高い。
- PC ↔ モバイル のリサイズ時に表示パネルを再制御するロジックも実装済み。

**軽微な懸念点**:
- 縦書きプレビューは横スクロール（overflow-x: auto）で列を確認するが、モバイルのスワイプ操作とページスクロールが競合する場合がある。明示的な「横スクロールで列を確認できます」の案内があると親切。

---

### 6. 信頼性（プライバシー説明・サンプルテキストの著作権問題がないか）

**評価: ✅ 良好**

- 「プライバシーについて」セクションで「テキストはブラウザ内のみで処理、外部サーバーへの送信なし」を明記しており、小説原稿という個人の創作物を扱うツールとして必須の説明が揃っている。
- 「縦書きプレビューとは」セクションでも同様の説明を繰り返しており、ユーザーへの安心感が高い。
- サンプルテキスト（夏目漱石「吾輩は猫である」）は青空文庫収録作品で著作権保護期間終了（没後 70 年経過）のため著作権上の問題なし。
- footer にプライバシーポリシーへのリンクあり。
- AdSense スクリプトタグはコメントアウト済みで審査前にスクリプトが読み込まれない安全な状態。

---

### 7. コンテンツ（使い方・縦書き・原稿用紙換算の解説充実度）

**評価: ✅ 良好**

- 「使い方」（6 ステップ ol）は画面の各機能を網羅しており、初めて使うユーザーでも手順が明確。
- 「縦書きプレビューとは」はターゲットユーザーの用途（カクヨム・なろう・同人誌）を具体的に記載しており、SEO テキストとしても機能する。
- 「原稿用紙換算（400字詰め）について」は換算ロジック（実質文字数 ÷ 400 の切り上げ）まで説明されており、数値に疑問を持ったユーザーが解消できる。
- 「ルビの入力方法」はコードサンプル付きで具体的。

**軽微な懸念点**:
- カクヨム・なろうの縦書き表示と本ツールのプレビューが完全一致するわけではない旨の断り書きがあると過度な期待を防げる（各サービス固有のフォントやレイアウトが異なるため）。

---

### 8. 広告・アフィリエイト（ad-unit配置・CSSに不要なスタイルがないか・アフィリエイトブロック）

**評価: ✅ 良好**

- `ad-unit` の CSS は `margin: 1.25rem 0;` のみで、空のブロックに対して余計な `min-height`・`background`・`border` 等は一切付与されていない。AdSense 審査時に空のスペースが目立つ問題なし。
- アフィリエイトブロック（ConoHa WING）は `PR` ラベルを明示しており、ステルスマーケティング規制（景品表示法 5 条 3 号）への対応が適切。
- アフィリエイトリンクに `rel="nofollow"` 付与済みで SEO リスクなし。
- アフィリエイトリンクの `min-height: 44px` + `line-height: 44px` でタッチターゲット確保済み。
- AdSense スクリプトはコメントアウト状態で、審査用の準備済みスロット（`id="ad-top"`, `id="ad-result"`）のみが存在する状態は適切。

| # | 優先度 | 内容 |

| 1 | High | `applyCharsPerLine()` の height 計算が `chars=40` + `fontSize=18` 等の組み合わせで `scroll-outer` 高さを超過し、縦書きテキストの下部が切れる可能性。height の上限を `tategumi-scroll-outer` の高さ（500/560px）に制限する処理を追加する。 |
