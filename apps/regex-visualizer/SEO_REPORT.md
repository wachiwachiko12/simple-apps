## 競合分析

| サイト | 種別 | 強み | 弱点 |
|-------|------|------|------|
| **Regexper.com** | 海外専門ツール | Railroad Diagram の元祖・老舗、図の品質が高い、OSSで信頼性高い | **UI が英語のみ**、説明文・ラベル一切日本語なし、モバイル最適化が弱い |
| **regex101.com** | 海外専門ツール | 多言語エンジン対応（PCRE/JS/Python等）、マッチング詳細解説、デバッガー機能 | **日本語UI非対応**、鉄道図（Railroad Diagram）機能なし、情報量が多すぎて初心者に難解 |
| **Debuggex.com** | 海外専門ツール | Railroad Diagram 対応、JS/Python/PCRE対応 | **英語UIのみ**、更新頻度が低く機能が古い、モバイル非対応 |
| **Regulex（github.io）** | OSSツール | JS正規表現の可視化に特化、シンプル | **日本語文字（マルチバイト）を \uXXXX にエスケープ**してしまう致命的な欠点、更新停止気味 |
| **Regexpra** | 日本語ツール | 日本語説明あり、スマホ対応 | Railroad Diagram 非対応、機能がシンプルすぎる、サイト情報が少ない |
| **digtools regex-visualizer** | 日本語ツール | 日本語UI、Railroad Diagram 対応、SVGエクスポート、サーバーレス処理 | 知名度が低い、コンテンツ（使い方説明・チュートリアル）が薄い |
| **Toolpods 正規表現テスター** | 日本語ツール | 日本語UI、スマホ対応、ログイン不要 | 可視化（図）機能なし、マッチング確認のみ |

### 競合強度サマリー

- **「正規表現 可視化」メインKW**: 中程度。海外ツール（Regexper/Debuggex）が上位だが**英語UIのみ**という明確な弱点あり。
- **「鉄道図 正規表現」**: 弱〜超弱。上位は個人ブログ・Atomプラグイン紹介記事が多く、専用の日本語ツールサイトがほぼ不在。
- **「regex visualizer 日本語」**: 弱。digtools が存在するが知名度低く、コンテンツが薄い。
- **「正規表現 図解」**: 中程度。解説記事（Qiita・Zenn）が上位に多いが、ツールとしての競合は限定的。

## 差別化戦略

### 勝てるポイント

1. **日本語UI × Railroad Diagram の組み合わせ**: Regexper/Debuggex は Railroad Diagram の品質が高いが英語のみ。日本語で同等品質を提供できるサービスは極めて少ない（digtools のみが近いが知名度低い）。
2. **日本語マルチバイト文字の正確な処理**: Regulex の「日本語を \uXXXX にエスケープする」欠点を回避できれば、日本語エンジニアに強く刺さる差別化になる。
3. **モバイル対応**: 海外主要ツール（Debuggex・Regulex）はモバイル対応が弱く、スマホで確認したいエンジニアのニーズを取りこぼしている。
4. **初心者向け解説コンテンツの添付**: 図の下に「この正規表現の読み方」を日本語で補足するコンテンツがあれば、Know 型（情報型）検索も取り込める。
5. **QAエンジニア・テストエンジニア向け特化訴求**: 競合はほぼ開発者向け汎用ツール。QA視点（テスト文字列の網羅性確認など）に特化した訴求で差別化可能。

### 実装すべき機能・コンテンツ

- 正規表現の各要素（量指定子・キャプチャグループ・文字クラス）の日本語説明ポップアップ
- 代表的な正規表現パターン集（メールアドレス・電話番号・郵便番号・日付など）をプリセットとして提供
- 「この図の読み方」日本語チュートリアルセクション
- よくある正規表現エラーの解説コンテンツ（SEO的にも有効）

## 総評・勝算

**競合強度: 中程度 / 勝算: 高**

「正規表現 可視化」カテゴリは、老舗の海外ツール（Regexper・regex101・Debuggex）が上位を占めるものの、**日本語UI を提供しているツールは極めて少ない**。Railroad Diagram 対応かつ日本語対応の専用ツールは調査時点で digtools のみであり、知名度・コンテンツ量ともに低い状態。

keisanlab の強みであるドメイン蓄積・日本語特化・モバイルファースト設計を活かせば、「正規表現 可視化 日本語」「鉄道図 正規表現」「regex visualizer 日本語」などのキーワードで十分に上位表示が狙える。

特に「鉄道図 正規表現」「正規表現 フローチャート」は競合がほぼ個人ブログのみで、**今すぐ狙える超弱競合ニッチ**。Railroad Diagram の日本語解説コンテンツを充実させることで、情報型検索（Know 型）からツール利用（Do 型）へ誘導するコンテンツ戦略が有効。

**優先アクション**:
1. title/h1/meta description に「鉄道図」「Railroad Diagram」「可視化」を必ず含める
2. 正規表現プリセット集（メールアドレス・電話番号・日付等）を追加してコンテンツ量を増やす
3. 「Railroad Diagramの読み方」日本語解説セクションをページ下部に追加
