### KW選定の考え方

- **「文章チェックツール 無料」「文章 読みやすさ チェック」** は検索ボリュームが最大で、Do型（実行型）の検索意図に直結する最優先KW
- **「可読性スコア 日本語」** は競合が極めて少ないニッチKW。数値スコアを出すツールが他にほぼ存在しないため、このKW単独での上位表示が狙いやすい
- **「漢字密度 チェック」「平均文長 チェック」** はロングテール。LUFT文字数カウンターが一部対応しているが専用ツールとしての訴求力は弱く、差別化余地がある

上位を占めるのは個人開発サイト・文字数カウンター・有料寄り校正ツールであり、**「可読性をスコアとして数値で出す無料の日本語専用ツール」は実質的に空白地帯**。

## 差別化戦略

### 勝てるポイント

1. **スコアの数値化**: 既存競合はハイライト表示や比率表示にとどまる。100点満点など直感的なスコアを出すことで「客観的評価」という訴求ができ、他に類似ツールがない
2. **複合指標の提示**: 漢字密度・平均文長・ひらがな比率・1文あたり句読点数を組み合わせて一画面で見せるUIは現状どのツールも実装していない
3. **改善ヒントの提示**: スコアが低い場合の改善アドバイス（「漢字が多すぎます → ひらがな・カタカナへの置き換えを検討」等）を自動表示することで差別化
4. **モバイルファースト**: so-zou.jp・読みやすさチェッカーともスマホUXが弱い。スマホでの利用を前提とした設計で優位に立てる
5. **利用シーン訴求**: ブロガー・ライター・就活生・社内文書作成者など具体的なペルソナ別の説明文をページに組み込むことでロングテールKWを自然に拾える

### 実装すべき機能（優先順）

1. 可読性スコア（0〜100点）のリアルタイム表示
2. 漢字密度（%）・ひらがな率・平均文長・長文（60字超）の文数 の統計パネル
3. スコアに基づく改善アドバイスの自動生成
4. ペルソナ別の「使い方」セクション（ブロガー・ライター・ビジネス文書）

## 実装推奨（app-builder 向け）

- **title タグ案**: `日本語文章の可読性スコアを無料チェック | 文章読みやすさ診断ツール`
- **meta description 案**: `文章を貼り付けるだけで可読性スコアを即座に算出。漢字密度・平均文長・ひらがな率を数値化し、読みやすい文章かどうかを無料で診断できます。ブロガー・ライター・ビジネス文書作成に。`
- **h1 案**: `日本語文章の可読性スコアを無料でチェック`

- 読みやすい文章の書き方
- 漢字の使いすぎ チェック
- 一文が長い 見直し
- ブログ記事 読みやすさ
- ライター 文章品質 チェック

## 総評・勝算

**競合強度: 低〜中 / 勝算: 高**

「可読性スコアを数値で出す無料の日本語ツール」というポジションは現時点でほぼ空白。既存競合は校正（誤字・文法チェック）に特化しており、「読みやすさの客観的スコアリング」を前面に出したツールは存在しない。

jReadability Portal は学術的根拠があるが UX が研究者向けであり、一般ユーザーが求める「貼り付けてすぐスコアがわかる」体験を提供できていない。

検索ボリューム単体は大きくないが、「文章チェックツール 無料」系のKWは合計で月間5,000〜10,000件程度の流入が見込める。ロングテールが多数存在し、内部リンク構造を整えることで複数KWでの上位表示が狙える。

AdSense 観点では、ライター・ブロガー・就活生・ビジネスパーソンが対象ユーザーとなるため、文章作成ツール・ライティング講座・転職サービス系の広告との親和性が高く、クリック単価（CPC）は比較的高め（推定 80〜150円）と想定される。
