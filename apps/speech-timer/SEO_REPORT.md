### 競合強度の総合評価

- **計算系（静的）ツール**: 多数存在・競合中程度。「入力→時間計算」で完結し、**実際に音読して練習する機能を持つサイトはほぼ皆無**。
- **純タイマー系**: presentimer.github.io など。原稿入力・話速計測なし。
- **スマホアプリ系**: 発話スピード計測アプリ（iOS）が近い機能を持つが、ブラウザWebツールとしては競合しない。
- **ブラウザ完結で「原稿ペースト + リアルタイム音読 + 話速計測」を同時に実現するWebツール**: 調査範囲では存在せず。

## 差別化戦略

### 勝てるポイント

1. **「計算」ではなく「練習」に特化している唯一のWebツール**
   - 競合はほぼすべて「文字数を入力→所要時間を計算」で終わる静的ツール。実際に音読しながらリアルタイムで話速を計測するツールはブラウザWebアプリとして存在しない。

2. **原稿ペースト → 制限時間設定 → 音読開始 → 話速リアルタイム表示という一気通貫のフロー**
   - ユーザーが求めているのは「自分の話すペースが合っているか確認しながら練習すること」。このニーズに正面から応える。

3. **完全無料・登録不要・ブラウザ完結**
   - スマホアプリ（発話スピード計測）はインストールが必要。本ツールはURLを開くだけで即使える。

4. **日本語特化**
   - 日本語の話速目安（300文字/分）を基準に、読み上げ速度を色分り表示するなど、日本語話者に最適化した UI を提供できる。

5. **競合の弱点を突く機能**
   - presentimer.github.io: 原稿入力なし → 本ツールは原稿表示しながら練習
   - digtools: プロンプターあるが話速リアルタイム計測なし → 本ツールは計測付き
   - 計算系全般: 練習機能ゼロ → 本ツールは実際の練習をサポート

### 実装すべき機能（app-builder 向け）

- 原稿テキストエリア（ペーストで即使用）
- 制限時間設定（分:秒）
- 音読スタート/ストップボタン
- 経過時間・残り時間のリアルタイム表示
- 話速リアルタイム計測（文字/分）+ 目安との差異表示
- 残り時間に応じた色変化（緑→黄→赤）
- 「300文字/分」を基準とした話速インジケーター
- 音読中のプロンプター（自動スクロール）

## 総評・勝算

**競合強度: 低〜中（狙い目）**

「原稿テキストをペーストして、実際に音読しながら話速をリアルタイムで計測できる」ブラウザWebツールは、2026年5月時点で調査範囲内では存在しない。競合は全て「静的な時間計算ツール」か「タイマーのみ」に留まっており、機能的に明確な空白地帯がある。

ターゲットユーザー（プレゼン直前の練習者・学生・就活生・登壇者）が実際に必要としているのは「計算」ではなく「練習」であり、この点で本ツールは既存競合に対して圧倒的な差別化が可能。

キーワード単体の月間検索数は各1,000〜3,000程度と大きくはないが、複数KWのロングテール合計で月間5,000〜10,000 PV程度の獲得が現実的。「音読タイマー」「スピーチ タイマー 無料」「プレゼン 話速 計測」は競合が弱く、新規サイトでも3〜6ヶ月で上位表示が見込める。

**勝算: 高い。早期投入を推奨。**
