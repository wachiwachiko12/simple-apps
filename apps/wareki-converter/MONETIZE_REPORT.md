### 必須修正事項

**問題1: 広告ユニットが空divのみで実際の広告コードが存在しない（最重要）**

`id="ad-top"` / `id="ad-result"` / `id="ad-table"` の3つのdivが存在するが、AdSenseの `<ins class="adsbygoogle">` タグが一切挿入されていない。現在、広告は一切表示されていない状態。

```html
<!-- 現状（広告が表示されない空のdiv） -->
<div class="ad-unit" id="ad-top" aria-hidden="true"></div>

<!-- 修正後（AdSense広告ユニットを挿入） -->
<div class="ad-unit" id="ad-top">
  <ins class="adsbygoogle"
       style="display:block"
       data-ad-client="ca-pub-6046047693794575"
       data-ad-slot="XXXXXXXXXX"
       data-ad-format="auto"
       data-full-width-responsive="true"></ins>
  <script>(adsbygoogle = window.adsbygoogle || []).push({});</script>
</div>
```

- 優先度: **必須**（これが未修正の場合、収益はゼロのまま）
- `aria-hidden="true"` は広告コード挿入時に削除すること（スクリーンリーダーへの影響を防ぐためAdSenseの自動処理に委ねる）

**問題2: ad-top の配置がユーザー離脱を誘発している**

`ad-top` はタブUIの直前（ツール入力UIの真上）に配置されている。ユーザーがページを開いた瞬間に広告エリアが視野に入り、ツール本体へのアクセスが一瞬遅れる構造になっている。直帰率100%の一因として疑われる。

詳細は「3. 直帰率100%の原因分析」を参照。

## 3. 直帰率100%の原因分析

### 構造的問題の特定

直帰率100%は以下の複合要因によって発生していると推定される。

**要因A（最大要因）: 広告が表示されていない**

現状では広告コードが存在しないため、ページ内にAdSenseコンテンツが何も表示されない。ユーザーの体験としてはツールを使い、結果を得て、即座に離脱するという完結型パターンになっている。これ自体は「変換ツール」の性質上ある程度は避けられないが、広告が表示されていれば「結果確認後に広告をちらりと見て離脱」という流れでインプレッションが発生する。

**要因B: タブUI構造のためスクロールが発生しない**

現在の設計はタブ切り替え型であり、ファーストビュー内でツールの主要機能（西暦→和暦変換）が完結する。ユーザーはスクロールせずにタスクを完了できるため、ページ下部の `ad-result` / `ad-table` / `howto-section` / `affiliate-block` まで到達しない。

**要因C: ad-top の位置が広告視認前離脱を誘発**

ページトップの広告枠は「広告が表示されたことでページの品質に不安を感じた離脱」を引き起こすリスクがある。ただし現状は広告コードが存在しないため、実際には空白エリアが表示されているだけとなり、これがレイアウトの違和感につながっている可能性もある。

**要因D: 内部リンクが下部にしか存在しない**

関連ツールへのリンク（確定申告シミュレーター・家計簿アプリ）および A8.net アフィリエイトリンクは `howto-section` の末尾に配置されており、直帰ユーザーには一切視認されていない。

## 4. 収益最適化施策（優先順）

### 施策1: 広告コード（`<ins>`タグ）の挿入（必須・即実施）

- 期待効果: 収益 0円 → 発生開始（前提条件）
- 実装コスト: 低
- 対象: ad-top / ad-result / ad-table の3枠すべて
- 注意: AdSense アカウントの管理画面で3つの広告ユニットを作成し、それぞれの `data-ad-slot` を取得して挿入する

### 施策2: ad-top をタブUIの下（最初の変換結果表示エリアの直下）に移動（推奨）

- 期待効果: CTR +15〜30%（ユーザーが目的達成直後に広告を視認するタイミングに最適化）
- 実装コスト: 低
- 根拠: `revenue-optimizer` 原則2「ユーザーが目的を達した後に広告を見せる」に合致。現状のタブUIの真上配置は「目的達成前」に広告が挟まるため視認効率が低い

```html
<!-- 修正前の順序 -->
<div class="ad-unit" id="ad-top">  ← タブの上（目的達成前）
<div class="tabs">
<section id="tab-seireki-to-wareki">
  <div class="result" id="result-s2w">  ← 変換結果表示

<!-- 修正後の順序 -->
<div class="tabs">
<section id="tab-seireki-to-wareki">
  <div class="result" id="result-s2w">  ← 変換結果表示
<div class="ad-unit" id="ad-top">  ← 結果の直下（目的達成直後）
```

### 施策3: 結果表示時にスクロールを促すコンテンツを追加（推奨）

- 期待効果: スクロール率 +20〜40%、ページ内滞在時間増加、下部広告視認率向上
- 実装コスト: 中
- 具体策: 変換結果の下に「この年の出来事」など簡易コンテンツを追加するか、「年齢早見表タブも確認してみる」のような誘導テキストを設ける

### 施策4: 内部リンク（関連ツール）をタブUI直下に追加（推奨）

- 期待効果: 直帰率 -10〜20%（他ツールへの回遊を促進）
- 実装コスト: 低
- 具体策: 現在ページ末尾にのみ存在する関連ツールリンクを、ツールUI直下（`ad-result` の前後）にもコンパクトに配置する。直帰ユーザーの視野に入るファーストビュー〜スクロール1段階の範囲が有効

```html
<!-- ad-result の直後に追加 -->
<div class="related-mini">
  <span>関連ツール：</span>
  <a href="/apps/freelance-tax-simulator/">確定申告シミュレーター</a>
  <a href="/apps/household-budget/">家計簿アプリ</a>
</div>
```

### 施策5: アフィリエイトリンクをより上部に引き上げ（情報提供）

- 期待効果: アフィリエイト収益の発生可能性向上
- 実装コスト: 低
- 現状: A8.net（お名前.com・A8.net自体）のアフィリエイトリンクがページ末尾にのみ存在
- 改善案: `howto-section` の冒頭（関連ツールリンクの直前）に移動するか、「和暦変換ツールをWebサイトに活用するなら」という文脈でさらに自然な位置に配置する
- 判断はユーザーに委ねる（ポリシー上の問題はなし）
