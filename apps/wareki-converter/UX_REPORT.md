# UX レビューレポート — 和暦・西暦変換ツール（wareki-converter）

**レビュー日**: 2026-07-23（検証レビュー・第2版）
**レビュアー**: ux-reviewer（ユーザー代表）
**使用ペルソナ**: B（個人事業主の飲食店オーナー・45歳男性）・D（家計管理したい主婦・38歳女性）・A（フリーランスデザイナー・32歳女性、モバイル動線の補助確認用）

**背景**: 前回レビュー（2026-07-23 第1版）で検出したブロッカー2件（B-1: デフォルトタブの方向違い／B-2: ページロード時に結果が空欄）・要改善5件（I-1〜I-5）について、app-builderが修正を実装したとの報告を受け、index.html / script.js / style.css を実地に再読み込みして「指摘事項が実際に解消されているか」を1件ずつ検証した。

**検証結論**: **前回指摘した7件すべてが実装レベルで解消されていることを確認した。** 新規に致命的な問題は見つからなかった。

## 総合判定

✅ 承認

> 開いた瞬間に「和暦→西暦」タブが表示され、しかも令和8年→2026年の変換結果がすでに入った状態で表示される。前回感じた「動いているのか分からない」「欲しい入力欄がどこにあるか分からない」という離脱要因が両方とも消えている。年齢早見表もタブの2番目に繰り上がり、プライバシー説明もツールのすぐ上に常時表示になった。細かい改善の余地はまだあるが、リリースを止める理由はない。

## 検証結果（前回指摘事項の再確認）

### 🚫→✅ B-1: デフォルトタブの方向違い — 解消を確認

- **場所**: index.html L76-79
- **検証**: タブの並びが `和暦 → 西暦`（`class="tab active"` / `aria-selected="true"`）→ `年齢早見表` → `年齢計算` → `西暦 → 和暦` に変更されている。対応する `#tab-wareki-to-seireki` セクション（L85-105）に `hidden` クラスが付いておらず、初期表示状態で正しく見えている。`#tab-seireki-to-wareki`（L156）側に `hidden` が付与されており、表示/非表示の入れ替えが正しく行われている。
- **ペルソナB視点**: 「開いてすぐ『元号』と『年』を入れる欄が出てきた。これなら昭和60年が何年か迷わず調べられる」
- **判定**: ✅ 解消

### 🚫→✅ B-2: ページロード時に結果が空欄 — 解消を確認（ロジック検証済み）

- **場所**: script.js L235-242
- **検証**: `DOMContentLoaded` の末尾で以下を実行している。
  ```js
  const reiwaYear = currentYear - ERAS.find(e => e.key === 'reiwa').start + 1;
  document.getElementById('era-select').value = 'reiwa';
  document.getElementById('wareki-input').value = reiwaYear;
  document.getElementById('btn-w2s').click();

  document.getElementById('seireki-input').value = currentYear;
  document.getElementById('btn-s2w').click();
  ```
  `currentYear`（L115で `new Date().getFullYear()`）を用いて `reiwaYear = 2026 - 2019 + 1 = 8` を算出し、`era-select`/`wareki-input` にセットした上で `btn-w2s` のクリックイベントを能動的に発火させている。ボタンのイベントリスナーは同一 `DOMContentLoaded` 内で先に登録済み（L162-180）のため、`.click()` は正しくハンドラを実行し、`warekiToSeireki('reiwa', 8)` → `2019+8-1=2026` を返し、`showResult` が同期的に呼ばれて `el.style.display='block'` になる。結果として初期表示から「西暦 2026 年」「令和8年 = 西暦 2026 年」が見える。
  同様に `seireki-input` にも当年をセットし `btn-s2w` を発火。現在非表示（`hidden`）の「西暦→和暦」タブの結果も裏側で事前計算されており、ユーザーがタブを切り替えた瞬間には既に答えが出ている（前回指摘の必要最低限を上回る実装）。
- **ペルソナD視点**: 「開いた瞬間に『令和8年＝2026年』って出てる。ちゃんと動いてるって分かって安心した」
- **判定**: ✅ 解消（ロジック上、算出値・表示条件とも妥当）

### ⚠️→✅ I-1: 年齢早見表の露出不足 — 解消を確認

- **場所**: index.html L76-79
- **検証**: タブ順が「和暦→西暦／年齢早見表／年齢計算／西暦→和暦」に変更されており、前回提案した順序と一致。年齢早見表が2番目に繰り上がった。
- **判定**: ✅ 解消

### ⚠️→✅ I-2: プライバシー説明がファーストビュー付近にない — 解消を確認

- **場所**: index.html L82
- **検証**: `<p class="privacy-note">🛡 入力データはブラウザ内でのみ処理し、サーバーには送信されません</p>` がタブ直下・ツールセクションの直前に常時表示で追加されている。スクロールなしで視認できる位置。
- **判定**: ✅ 解消

### ⚠️→✅ I-3: `.era-btn` のタッチターゲット不足 — 解消を確認

- **場所**: style.css L427-441
- **検証**: `.era-btn { ... min-height: 44px; touch-action: manipulation; ... }` に修正されている（旧: 36px）。同ページの `.tab`（L91）・`button`（L190）・`.hint-link`（L247）と並び、全クリック要素が44px以上・`touch-action: manipulation` を満たしている。
- **判定**: ✅ 解消

### ⚠️→✅ I-4: 文脈のずれたアフィリエイト広告 — 解消を確認

- **場所**: index.html 全体（該当セクション削除済み）
- **検証**: 前回指摘の「Webサイト・ブログを始めるなら」（お名前.com・A8.net紹介）セクションは grep で検索しても該当箇所なし。完全に削除されている。代わりに `related-mini`（L173-177）・`related-links`（L266-269）として「確定申告シミュレーター」「家計簿アプリ」という利用文脈に沿った自社ツールへの誘導のみが残っており、唐突感がない。
- **判定**: ✅ 解消

### ⚠️→✅ I-5: 結果表示後の次アクション誘導なし — 解消を確認

- **場所**: script.js L43-50（`showResult` の `hint` 引数）、L158（s2w）、L179（w2s）、L214（age-calc）
- **検証**: 各変換結果の下に `.result-hint` として関連タブへの誘導リンク（例:「この年の年齢を知りたい方は年齢計算タブへ →」）が `data-goto-tab` 属性付きで挿入され、L229-233 のイベント委任で `gotoTab()` を呼び出しクリック一つで該当タブへ遷移する。`.hint-link` は `min-height: 44px` でタップしやすい（style.css L243-261）。初期表示時にも自動計算結果と一緒にこのヒントが出るため、着地直後から次の導線が見える。
- **判定**: ✅ 解消

## 追加確認事項（今回のレビュー範囲）

### AdSense広告ユニット

- **場所**: index.html L107-114（ad-top）、L164-171（ad-result）、L192-199（ad-table）
- **確認**: 3箇所すべてに `<ins class="adsbygoogle" style="display:block" data-ad-client="ca-pub-6046047693794575" data-ad-slot="" data-ad-format="auto" data-full-width-responsive="true"></ins>` が正しく挿入されており、`aria-hidden` は付与されていない（アクセシビリティツリーから除外されておらず問題なし）。
- **付随メモ（ブロッカーではない）**: `data-ad-slot=""` が空のままで、MONETIZE_REPORT.md 側でも「AdSense管理画面で実際のスロットIDを取得して挿入すること」と既にフォロー事項化されている。UX上は空スロットでもレイアウト崩れは起きないため今回は要改善扱いとしないが、本番公開前に monetize-reviewer 側のチェックリストで確実に潰すこと。

### 新規コンテンツ（プライバシー注記・関連ツールリンク・結果下ヒント）のデザイン整合性

- `.privacy-note`（グレー背景・アイコン付き1行）、`.related-mini`（カード型・関連ツール2件）、`.result-hint`（結果ブロック内に区切り線付きでリンク表示）はいずれも既存のCSS変数（`--primary` / `--radius-sm` / `--border` 等）を使った配色・角丸・余白で統一されており、後付けで浮いた印象はない。フォントサイズも周辺要素（0.8125rem〜0.9rem）と揃っている。
- 唐突さ・過剰演出は見られず、テンプレートの意匠と整合している。
