# UX レポート — タイピング練習ツール

**総合判定: ⚠️ 要修正（修正済み）**

レビュー日: 2026-05-26
ペルソナ: 「e-typingが古くてスマホ非対応で不満を持つ20〜40代のビジネスパーソン。スマホとPCの両方でタイピング速度を伸ばしたい。登録なしで今すぐ使いたい。」

---

## 1. ファーストビュー

**評価: ✅ 良好（修正後）**

- ヘッダーに「ローマ字・かな・英語に対応 | WPM・正確率をリアルタイム計測」と明記されており、サービス内容が一目でわかる
- 3モードタブ（英語・ローマ字・かな）がページ最上部に配置され、視覚的に切り替えられる
- 初期状態のテキストエリアに「① モード・テキストを選ぶ → ② スタートボタンを押す → ③ タイピング開始！」と手順が表示され、操作フローが明確になった（修正済み）
- e-typingの代替として登録不要・無料と meta description に記載されており、流入ユーザーの期待に応えている

**修正前の問題:** 「スタートボタンを押して練習を開始してください」のみで、モードやカテゴリを先に選ぶべきか不明瞭だった。

---

## 2. 入力UX

**評価: ✅ 良好（一部修正済み）**

- 文字ハイライトは正解＝緑・ミス＝赤背景で非常に分かりやすい
- 現在位置は青いアンダーラインとブリンクアニメーションで明確
- スタートボタン押下後に入力フィールドが有効化され、自動フォーカスされる
- Backspace無効化（削除禁止）により打ち直しなしで集中できる
- ミス時のシェイクアニメーションで即時フィードバックがある

**修正した問題:**
- モード/カテゴリ切り替え時に練習中でも状態が変わり不整合が生じていた → 切り替え時に自動でリセットするよう修正
- かなモードでBackspaceが完全に無効化されていたため、IME変換中の文字修正が不可能だった → かなモード中はBackspaceを許可するよう修正

---

## 3. 結果表示

**評価: ✅ 良好**

- 完了モーダルにWPM・正確率・ミス数・タイムの4指標が2×2グリッドで表示され見やすい
- ランク表示（S/A/B/C/D）がゴールドで大きく表示され達成感がある
- 新記録時は「新記録！」バッジがポップアニメーションで出現する
- 「もう一度」「閉じる」ボタンが明確
- ランクの基準テーブルが使い方セクションに掲載されており、自分のレベルが把握できる

**修正した問題:** ランク基準がWPMのみ英語モード向けだったため、ローマ字・かなモードはWPMが低くなる旨の注釈を追加した。

---

## 4. エラー処理

**評価: ✅ 問題なし**

- スタートボタンを押すまで入力フィールドは `disabled` 状態で `cursor: not-allowed` が表示される
- placeholder テキスト「ここに入力してください（スタートボタンで開始）」でスタート前の状態を明示
- テキストが選択されていない状態でのスタートは存在しない設計（テキストはランダムに自動選択される）
- モード・カテゴリの整合性チェックがあり、英語モードで日本語カテゴリを選ぼうとすると自動調整される

## 5. モバイル

**評価: ✅ 良好**

- `maximum-scale=1.0` あり（iOS での意図しないズーム防止）
- モードタブ: `min-height: 44px` — タッチターゲット基準クリア
- カテゴリ選択 select: `min-height: 44px` — クリア
- スタート/リセットボタン: `min-height: 48px` — クリア
- 戻るリンク: `min-height: 44px`、`line-height: 44px` — クリア
- 入力フィールド: `min-height: 48px` — クリア
- タイピング表示エリアのフォントサイズはモバイルで `1.25rem`、`≥640px` で `1.4rem` とレスポンシブ対応
- モバイルでのタイピングは実用的には難しいが、スマホで使う場合のスクロールや操作ターゲットは十分確保されている

## 8. 広告

**評価: ✅ 問題なし**

- `.ad-unit` には `margin` のみ設定されており、`min-height` や `background` は未設定（AdSenseポリシー準拠）
- 広告配置: ツール上部（ad-top）・使い方セクション直前（ad-result）の2か所
- `aria-hidden="true"` で スクリーンリーダーから除外されている
