# UX_REPORT — 会議コスト計算ツール

**レビュー日:** 2026-05-21  
**レビュア:** UXレビューエージェント（ux-reviewer）  
**ペルソナ:** 会社員またはマネージャー。「この会議、コスト大丈夫？」と感じていて実際の金額を見てみたい人。スマホで会議中に使うことも想定。ITリテラシーは普通。

---

## 総合判定

## ⚠️ 要修正

ブロッカー級の問題はないが、**空のグレーブロック（ad-unit）** と **削除ボタンのタッチターゲット不足** の2点は早期修正を推奨する。その他、UX改善により直帰率低下と滞在時間増加が見込める。

---

## 観点別評価

### 1. ファーストビュー ✅ 良好

- h1「会議コスト・ROIリアルタイムカウンター — 今この瞬間も積み上がる会議費用を可視化」は目的を一発で伝えている。
- カウンター表示（`¥0` / `00:00:00`）と「会議スタート」ボタンが最上部に固まっており、ページを開いた瞬間に操作できる構成は優秀。
- 「会議スタート」ボタンは `background: linear-gradient(135deg, #dc2626 0%, #f97316 100%)` で赤→オレンジのグラデーション。視認性・CTAとして十分。
- **軽微な問題:** ヘッダー直下に `.ad-unit`（後述）が挿入されており、広告が未表示のときにグレーブロックが表示されカウンターまでのスクロールが増える。

### 2. 入力UX ✅ 良好（一部改善余地あり）

- 職種セレクトには時給が `マネージャー（時給 4,808円）` のように表示されており、数値の根拠が入力時点で確認できる点は親切。
- 人数入力は `min=1 / max=50` で制限されており、過大入力を防げている。
- `参加者を追加` ボタンは `min-height: 44px` かつ全幅 (`width: 100%`) で操作しやすい。
- **問題（要修正）:** 削除ボタン `.btn-remove-participant` は `min-width: 32px / min-height: 44px` だが、実際に描画される横幅が 32px しかなく Apple HIG / Material の推奨 44px に届いていない。スマホで削除しようとして隣の人数入力欄をタップしてしまうリスクがある。`min-width: 44px` に修正が必要。
- **軽微な問題:** `.participant-hourly` が `×3名` のような簡易表示に留まっており、行ごとの小計コスト（例: `¥9,939/時`）が見えない。参加者が増えたときどのロールがコスト高かが分からず、コスト意識向上のストーリーとしてやや弱い。

### 3. 結果表示 ✅ 良好

- カウンター値 `counter-value` は `font-size: 3.5rem / font-weight: 800` で視認性が高く、タイマー稼働中は `.running` クラスでオレンジグローエフェクトが付く演出が「コストが積み上がる緊迫感」を演出している。
- `¥X / 分` 表示（`counter-rate`）で単位時間コストも確認できる。
- 早見表（30分/1時間/90分/2時間）は「会議前の判断材料」として有用。一人あたりコストも表示されており分かりやすい。
- ROI表示は `positive / negative / neutral` の色分けで直感的。`roi-percentage` の `font-size: 2rem` で数値も目立つ。
- **軽微な問題:** タイマーが停止中にROI入力欄の値を変更しても `updateROI()` は呼ばれるが、`elapsedSeconds === 0` のときは `roiResult` が空になる。会議前に「このメンバーで60分開いたらROI何%が必要か」を事前シミュレーションできると付加価値が高い（現状は未対応）。

### 4. エラー処理 ✅ 概ね良好

- 人数入力は `isNaN(v) || v < 1 ? 1 : 1` で最小値1に丸められる。
- 参加者0人（全員削除）で `calcCostPerSecond()` は `0` を返し、カウンター表示は `¥0` のまま正常動作する（タイマーは動くが0円なので実害なし）。
- localStorage の読み込み・書き込みは try/catch で囲まれており、プライベートブラウズでもクラッシュしない。
- **軽微な問題:** 参加者を全削除してもタイマーが起動できてしまう（¥0カウント）。参加者が0人のときは「会議スタート」を disabled にするか、警告トーストを出すと誤操作を防げる。
- **軽微な問題:** 数値入力で負の値を入力しようとするとブラウザのスピナーで `1` にクランプされるが、フォームのフォーカスアウト時に表示値が不整合になるケースがある（`input` イベントで処理しているため `blur` 時は再計算しない）。

### 5. モバイル対応 ⚠️ 要修正

- `maximum-scale=1.0` が設定されており iOS でのピンチアウト禁止ポリシーに対応している。
- 主要ボタンの `min-height` は以下の通り：
  - `btn-start / btn-stop / btn-reset`: `min-height: 48px` → ✅
  - `btn-add`: `min-height: 44px` → ✅
  - `affiliate-link`: `min-height: 44px` → ✅
  - `.btn-remove-participant`: `min-height: 44px` だが `min-width: 32px` → **⚠️ 横幅不足**
- `cost-table-wrapper` に `overflow-x: auto; -webkit-overflow-scrolling: touch` が設定されており横スクロールに対応している。
- `.participant-row select / input` は `min-height: 44px` → ✅
- `roi-form` はモバイルで縦並び（`flex-direction: column`）、640px以上で横並びに切り替わる。スマホ会議中の片手操作に適切。
- **問題（要修正）:** `.btn-remove-participant` の `min-width` を `44px` に修正することで Apple HIG 完全準拠となる。

### 6. 信頼性 ✅ 良好

- 各職種の時給がセレクトオプション内に表示されており「なぜこの金額か」が入力時点で分かる。
- 「会議コストの計算方法」セクションで年収÷2080時間という根拠を明示しており、信頼性が高い。
- 「機会コストベース」で実態はさらに高くなることも正直に注記されている。
- プライバシーについて「localStorageにのみ保存・外部送信なし」と明記されている。
- フッターにプライバシーポリシーリンクあり。

### 7. コンテンツ ✅ 良好

- 「使い方（6ステップ）」「会議コストの計算方法」「ROIとは」「会議コスト削減のヒント」「プライバシーについて」の5セクションが揃っており、AdSense審査に十分なテキスト量がある。
- ROI削減ヒントで Slack・ドキュメント非同期対応など具体的なアクションが書かれており、コンテンツとして完結している。
- **軽微な問題:** howto-section の `h2` が `<section>` の直下にあるが、構造上はアウトラインが `h1 → h2` ではなく `h1（header） → h2（howto内）` と飛んでいる。ツールセクションの `h2.section-title` と並列になっており、スクリーンリーダーユーザーには混乱を招く可能性がある。軽微だが `<section>` 毎の見出しレベル整合性を確認するとよい。

### 8. 広告・アフィリエイト ⚠️ 要修正（ad-unit 空ブロック）

- AdSense スクリプトはコメントアウト状態（`<!-- <script async ...> -->`）なので現時点で広告は配信されない。
- **問題（要修正）:** `.ad-unit` の CSS に `min-height: 90px; background: #e2e8f0;` が設定されている。AdSenseが未設定の状態では**中身が空のグレーブロック（90px高）が2箇所表示される**。ユーザーには壊れたレイアウトに見え、信頼性を損なう。AdSense本番運用前は `.ad-unit` に `display: none` を適用するか、`min-height` と `background` をAdSense有効時のみ付与する条件分岐が必要。
- アフィリエイトブロック（FREENANCE）は `PR` ラベル付きで景表法対応済み。`rel="nofollow"` あり。`min-height: 44px / line-height: 44px` でタッチターゲット十分。

## 修正優先度まとめ

| 優先度 | 箇所 | 問題 | 修正案 |
|--------|------|------|--------|
| 高 | `.ad-unit` CSS | 空のグレーブロックが2箇所表示される | AdSense本番化まで `display: none` を追加するか `min-height` / `background` を除去 |
| 高 | `.btn-remove-participant` | `min-width: 32px` でタッチターゲット44px未満 | `min-width: 44px` に変更 |
| 中 | 参加者0人でタイマー起動 | `¥0` カウントが起動できる | 参加者0人時は `btn-start` を `disabled` にする |
| 低 | `.participant-hourly` | 行ごとの小計コストが非表示 | `×3名（¥9,939/時）` のように小計を追記 |
| 低 | 見出しレベル | howto-sectionの `h2` がアウトライン上で並列 | セクション構造の整理を検討 |
