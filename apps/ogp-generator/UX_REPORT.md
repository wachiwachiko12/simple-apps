# UX レビューレポート — OGP カード作成ツール

**レビュー日**: 2026-05-21
**レビュアー**: UX Agent（一般ユーザー視点）
**ペルソナ**: ブログを運営しており、SNS シェア時のサムネイル画像（OGP）を作りたい人。デザインの専門知識はないが、ブラウザ操作には慣れている。スマホ・PC どちらからもアクセスする。

---

## 総合判定

## ⚠️ 要修正

重大なブロッカーはなく、コアな機能は動作するが、スマホ操作性と視覚的なフィードバック面でユーザーが「これでよいのか？」と迷う箇所が複数ある。修正後のリリースを推奨する。

---

## 観点別レビュー

### 1. ファーストビュー（h1・説明文・ツールの目的）

**評価: ✅ 良好**

- `<h1>` に「OGP・SNSカード画像ジェネレーター — シェア用サムネイルをブラウザで無料作成」と明記されており、ツールの目的が一目でわかる。
- サブタイトル「1200×630px のOGPカードをブラウザだけで作成・即ダウンロード。Twitter/X・Facebook・LINE対応。登録不要。」が具体的かつ簡潔で、ユーザーへの訴求力が高い。
- ヘッダー背景のグラデーションが視覚的なブランド感を演出しており、信頼感がある。
- PC 表示時の `h1` フォントサイズが `1.3rem` とやや小さく、パッと見の印象が弱い点は改善の余地がある（`1.5rem` 程度が望ましい）。

---

### 2. 入力 UX（テキスト入力・カラー選択・プリセット選択）

**評価: ⚠️ 要改善**

**良い点:**
- プリセットを選ぶだけでプレビューに即反映される点は、初心者に優しい設計。
- 文字カウンターがリアルタイムで更新され、残り文字が視覚的にわかる（黄色・赤の警告色付き）。
- テキスト・textarea・ラジオボタンすべてに `min-height: 44px` が設定されており、タッチターゲット基準を満たしている。

**問題点:**
- **プリセット選択時に入力内容が上書きされる点が無警告**: 既にタイトルや説明文を編集した後にプリセットボタンをクリックすると、入力内容が無言で消える。初心者が誤タップした際に「書いた文章が消えた」と感じるリスクがある。
  - 推奨対応: プリセットボタンにホバー時の tooltip「現在の入力内容がリセットされます」を表示するか、確認なしで上書きする旨を UI 上に一言添える。
- **カスタムカラー入力後に選択状態がどこにも表示されない**: プリセット色スウォッチの active 状態が全解除されるが、カスタムカラーピッカーのラベル（「+」ボタン）がアクティブ状態になる仕組みがないため、「何色が選択中か」が視覚的に消えてしまう。
  - 推奨対応: カスタムカラー選択後は `color-custom-label` に `active` スタイル（枠線強調など）を付与する。
- **絵文字入力欄の説明が不足**: `maxlength="4"` が設定されているが、「1文字まで」「複数入れてもよいか」がラベルだけでは判断できない。例示（「例: 📝」）はあるが、動作の説明がないと入力に迷う。

---

### 3. 結果表示（プレビューの見やすさ・ダウンロードボタン）

**評価: ✅ 良好（軽微な問題あり）**

**良い点:**
- Canvas による即時プレビューは直感的で、入力に連動してリアルタイム更新されるため「どんな画像が生成されるか」がすぐわかる。
- `aspect-ratio: 1200 / 630` で縦横比を維持しており、プレビューが実際の出力と同じ比率で表示される。
- ダウンロードボタンが設定パネル最下部に配置されており、操作フローが「設定 → ダウンロード」の順で自然。ボタンテキストも「PNG でダウンロード（1200×630px）」と出力サイズまで明記されていて親切。

**問題点:**
- **ダウンロード後のフィードバックがない**: ボタンをクリックしても見た目が変わらないため、「ダウンロードされたのか？」と不安になる。ブラウザ依存のダウンロード通知に頼り切っており、特にスマホでは気づきにくい。
  - 推奨対応: ボタンクリック後に一時的に「ダウンロード完了！」テキストや色変化を表示する。
- **モバイルでは設定パネルとプレビューが縦並び**: 設定を変更 → 下にスクロールしてプレビューを確認 → また上にスクロールして変更、という往復が生じる。スマホ利用者には地味にストレスになる。
  - 推奨対応: モバイル時にプレビューを設定パネルより先（上部）に表示するか、スクロールなしで確認できるスティッキーミニプレビュー（縮小表示）を上部に追加する。

---

### 4. エラー処理（空テキスト・不正カラーコード）

**評価: ⚠️ 要改善**

**良い点:**
- `drawWrappedText` 関数では `if (!text) return y;` による空テキスト判定がされており、空の場合は描画をスキップする安全な実装。
- `input[type="color"]` はブラウザ標準の color picker を利用しているため、不正カラーコードの入力自体が起こらない。

**問題点:**
- **全フィールド空欄でも何も起きない**: タイトル・説明文・サイト名がすべて空の場合、Canvas には装飾だけの空白画像がレンダリングされ、ユーザーが意図せず空画像をダウンロードしてしまうリスクがある。
  - 推奨対応: ダウンロードボタン押下時にタイトルが空の場合は「タイトルを入力してください」のアラートまたはインラインエラーを表示する。
- **プリセット初期値が日本語の指示文**: 「記事タイトルをここに入力してください」という文字列が初期状態のプレビューに表示され、プレビューを見た瞬間に「これは完成品ではなく入力例だ」とわかるが、初心者が「このまま使えばいいのかな？」と誤解してダウンロードしてしまう可能性がある。
  - 推奨対応: placeholder スタイルの薄い色でダミー表示し、実際の入力欄を空にしておくか、初期値は短いサンプルテキスト（「OGP サンプルタイトル」など）にとどめる。

---

### 5. モバイル対応（viewport・タッチターゲット・スクロール）

**評価: ✅ 良好（軽微な問題あり）**

**良い点:**
- `<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0">` が設定されており、iOS での入力フォーカス時の自動ズームが防止されている。
- ほぼ全インタラクティブ要素（ボタン・入力欄・ラジオラベル）に `min-height: 44px` が設定されている。
- モバイル時は `flex-direction: column` で縦並びレイアウトになっており、画面幅に適した表示になっている。

**問題点:**
- **カラースウォッチのタッチターゲットが 36px**: `.color-swatch` は `width: 36px; height: 36px` であり、44px の基準を下回っている。隣接するスウォッチ間の gap が `0.5rem` しかなく、スマホで誤タップが起きやすい。
  - 推奨対応: スウォッチ自体を 44px に拡大するか、周囲に透明なパディングを設けてタッチ領域を拡張する。
- **カスタムカラーラベルも 36×36px**: 同様に基準未達。
- **プレビューがスクロール下部に移動する問題**: 前述（観点3）の通り、モバイルでは設定変更とプレビュー確認の往復スクロールが必要。

---

### 6. 信頼性（プライバシー説明・データ非送信の明記）

**評価: ✅ 良好**

- 「プライバシーについて」セクションに「入力したテキスト・選択したデザインはすべてお使いのブラウザ内でのみ処理されます。外部サーバーへのデータ送信は一切行いません」と明記されており、信頼性確保の観点から十分な記載がある。
- フッターに「プライバシーポリシー」リンクがあり、サイト全体のポリシーにも誘導できている。
- HTTPS 配信前提のサイト構成で、ブラウザ上の Canvas API だけで完結する設計も安心感につながる。

---

### 7. コンテンツ（使い方・OGP 解説セクション）

**評価: ✅ 良好**

- 「使い方」に 6 ステップの番号付きリストがあり、WordPress でのアップロード方法まで解説されており、ターゲットユーザー（ブロガー）にとって具体的で実用的。
- 「OGP 画像とは？」セクションで OGP の概念・推奨サイズ・対応 SNS を簡潔に説明。知識ゼロのユーザーでも理解できる。
- 「OGP 画像を WordPress に設定する方法」が各 SEO プラグイン名（All in One SEO・Yoast SEO・SWELL・AFFINGER）まで言及しており、具体性が高い。
- AdSense 審査向けのコンテンツ充実度としても合格レベル。

**軽微な改善提案:**
- 使い方の手順とエディタが離れているため、ユーザーが使い方を読んでから上にスクロールして操作する流れになる。ツール上部に折りたたみ式の簡易ガイド（3 ステップ程度）を追加すると操作開始のハードルが下がる。

---

### 8. 広告・アフィリエイト

**評価: ✅ 良好（AdSense 未設定は要確認）**

**良い点:**
- `#ad-top`（ヘッダー直下）と `#ad-result`（結果下）の 2 箇所に広告ユニットが配置されており、コンテンツの邪魔にならない位置に収まっている。
- アフィリエイトブロックは `PR` ラベルを付け、ConoHa WING へのリンクは `rel="nofollow"` を付与しており、ステルスマーケティング規制・Google ポリシーへの配慮が見られる。
- アフィリエイトリンクの `min-height: 44px; line-height: 44px` 設定でタッチターゲットを確保している。

**確認事項:**
- AdSense スクリプトがコメントアウト（`<!-- <script async src="...ca-pub-XXXXXXXX"...> -->`）されたままである。本番デプロイ前に実際の Publisher ID を設定し、コメントアウトを解除する必要がある。
- 広告ユニット div に内容がない状態でも `min-height: 90px` でスペースが確保されており、AdSense 審査後の挿入準備は整っている。

## 修正優先度まとめ

| 優先度 | 問題 | 推奨対応 |
|--------|------|----------|
| 高 | カラースウォッチ（36px）がタッチターゲット 44px 未達 | サイズを 44px に変更またはタッチ領域を拡大 |
| 高 | ダウンロード後のフィードバックがない | ボタンに一時的な完了表示を追加 |
| 高 | 全フィールド空欄で空画像がダウンロードできてしまう | タイトル空欄時にバリデーションエラーを表示 |
| 中 | プリセット切替で入力内容が無警告上書き | ホバー tooltip または UI 上の注記を追加 |
| 中 | カスタムカラー選択後に視覚的選択状態が消える | `color-custom-label` にアクティブスタイルを追加 |
| 中 | モバイルでプレビュー確認に往復スクロールが必要 | プレビューを設定パネル上部に移動またはミニプレビュー追加 |
| 低 | PC での `h1` フォントサイズがやや小さい（1.3rem） | `1.5rem` 程度に拡大 |
| 低 | AdSense スクリプトがコメントアウトのまま | 本番デプロイ前に Publisher ID を設定して有効化 |
